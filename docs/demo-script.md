# LabelHub AI MVP 演示脚本

## 演示目标

本次演示要证明：
- 项目有完整的端到端闭环：创建 → 标注 → AI 预审 → 人工审核 → 统计
- 动态 Schema 可以驱动不同类型任务的标注表单
- Mock AI 预审可以作为人工审核前的辅助层
- 三角色流程清晰，职责分离
- Dashboard 可以实时反馈任务状态变化

## 演示准备

**方式一：本地启动**

```bash
# 终端 1：启动后端
cd backend
npm ci
npm run dev

# 终端 2：启动前端
cd frontend
npm ci
npm run dev
```

**方式二：Docker 启动**

```bash
docker compose up --build
```

**访问地址**：`http://localhost:5173`

**演示账号**：用户名填 `demo`，切换 admin / annotator / reviewer 三个角色

**建议准备**：系统预置了 3 条任务（情绪分类、商品标题质量判断、内容安全审核），可以直接演示。

## 3-5 分钟演示流程

### 0:00 - 0:30 项目定位

**操作**：打开登录页，停留 5 秒让评委看清楚。

**屏幕展示**：深色登录页，左侧有标题和 3 个功能亮点卡片（动态表单、AI 预审、三角色协作），右侧是登录表单。

**此时可以说**：

> "这是一个 AI 辅助数据标注审核的 MVP 原型。数据标注通常需要人工标注和质量检查，这个项目用简化的方式展示了一个完整闭环：管理员创建任务、标注员填写表单、系统做 AI 预审、审核员做最终判断。"

**这一段证明**：项目定位清晰，30 秒内能说清楚做什么。

## 评委 / 面试官可能提问与回答

### Q1：AI 是真的吗？

**回答要点**：

当前是基于规则的 mock 实现，不是真实大模型。它用关键词检测和答案长度做简单判断。它的价值在于展示 AI 预审层在产品流程中的位置——标注提交后、人工审核前，系统先给出结构化风险提示。接入真实 LLM 时需要把该模块改为异步 provider 调用，并补充密钥配置、错误处理、超时和回退策略。

### Q2：为什么用 JSON 文件存储？

**回答要点**：

MVP 阶段的取舍。用 JSON 文件可以降低部署复杂度，`docker compose up` 就能跑起来。数据访问逻辑集中在 `dataStore.js` 的 `readTasks` 和 `writeTasks` 两个函数，后续替换为数据库只需要改这个文件。

### Q3：这个项目和普通表单系统有什么区别？

**回答要点**：

普通表单通常是固定字段，提交后就结束了。本项目的核心是工作流：schema 驱动表单 → 标注结果接入 AI 预审 → AI 建议展示给审核员 → 审核员做最终判断 → 统计更新。它关注的是"标注质量如何保证"这个问题。

### Q4：如果要接真实大模型，怎么扩展？

**回答要点**：

把 `mockAIReview` 改造成异步 provider 层，调用 OpenAI-compatible API（DeepSeek、Ollama 等），同时调整调用方以处理异步结果。还需要处理配置与密钥、超时、失败回退、调用成本和日志。输出结构（riskLevel、confidence、possibleIssue、suggestion）可以保持不变。

### Q5：如果要接数据库，怎么扩展？

**回答要点**：

将 `dataStore.js` 替换为 repository / DAO 层。可以先用 SQLite 做本地开发，再迁移到 PostgreSQL。需要设计任务表、标注表、审核表、用户表。接口层不需要大改，因为数据访问已经封装在 dataStore 里了。

### Q6：最大技术亮点是什么？

**回答要点**：

动态 Schema + 三角色工作流 + AI 预审层的组合。单独做动态表单不难，单独做审核流程也不难，但把它们组合成一个可演示的完整闭环，并且保持接口设计的可扩展性，这是这个项目的价值。

## 演示失败备用话术

### 如果后端没启动

> "后端服务还没启动，我先启动一下。这是一个前后端分离的项目，后端是 Express 服务，前端通过 Vite 代理访问 API。"

### 如果前端请求失败

> "看起来 API 请求失败了，我检查一下后端状态。这个项目的前端通过 `/api` 代理到后端，如果后端没启动会报网络错误。"

### 如果任务状态不对

> "任务状态可能已经被之前的操作改变了。我先用 admin 角色看下 Dashboard 的统计，确认当前任务分布。"

### 如果 Docker 没运行

> "Docker 没有运行。这个项目支持两种启动方式：本地 Node.js 启动和 Docker Compose。Docker 方式更适合快速演示，本地方式更适合开发调试。"

### 如果页面没有任务

> "当前没有任务，我先用 admin 角色创建一条。系统预置了 3 条任务，但如果数据文件被清空，管理员可以随时创建新任务。"
