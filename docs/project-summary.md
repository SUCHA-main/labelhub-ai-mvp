# 项目总结

## 项目背景

数据标注是 AI 应用落地过程中的关键环节，但传统标注平台往往存在任务配置不灵活、质检反馈慢、审核链路割裂等问题。LabelHub AI MVP 面向比赛展示和原型验证，聚焦“快速创建任务、快速完成标注、快速看到 AI 辅助质检价值”。

## 解决的问题

- 标注任务配置成本高：通过动态 schema 让管理员无需改代码即可定义表单。
- 标注质量反馈滞后：标注提交后立即触发 AI 预审，给出结构化风险提示。
- 审核信息不集中：审核员在同一页面查看原文、标注结果和 AI 预审建议。
- 演示部署复杂：使用 JSON 文件存储和 Docker Compose，降低本地运行成本。

## 核心功能

- 简单登录，支持 `admin`、`annotator`、`reviewer` 三种角色。
- 管理员创建标注任务，任务包含标题、待标注文本和动态表单 schema。
- 标注员查看任务列表，按 schema 渲染表单并提交标注结果。
- 后端调用 `mockAIReview` 生成风险等级、置信度、问题提示和修改建议。
- 审核员查看待审核任务，选择通过或驳回并填写审核意见。
- Dashboard 展示任务状态、通过率、AI 风险提示数和待处理任务数。

## 技术架构

| 层级 | 说明 |
| --- | --- |
| 前端 | React + Vite + Tailwind CSS，负责角色工作台、动态表单和 dashboard 展示 |
| 后端 | Node.js + Express，提供登录、任务、标注、审核和统计 API |
| 存储 | JSON 文件保存任务数据，便于 MVP 演示和后续替换数据库 |
| 部署 | Docker Compose 编排前端 Nginx 容器和后端 Node.js 容器 |

## AI 预审设计

AI 预审以 `mockAIReview` 形式实现，模拟真实模型或规则引擎的输出。返回结构包括：

- `riskLevel`：风险等级，支持低、中、高。
- `confidence`：AI 对当前预审判断的置信度。
- `possibleIssue`：可能存在的问题提示。
- `suggestion`：给审核员或标注员的修改建议。

当前实现根据标注内容长度和风险关键词生成结果，后续可替换为真实大模型、规则引擎或多模型质检服务。

## 项目亮点

- 完整闭环：任务创建、标注提交、AI 预审、人工审核、统计看板一体化。
- 演示友好：内置情绪分类、商品标题质量判断、内容安全审核 3 类任务。
- 易扩展：动态 schema、文件存储和 mock AI 预审都可平滑替换为生产级组件。
- 易部署：本地 Node.js 启动和 Docker Compose 启动两种方式都支持。

## 后续计划

- 接入真实用户登录、权限控制和审计日志。
- 将 JSON 文件存储替换为 SQLite、PostgreSQL 或 MongoDB。

- 增加任务分配、批量导入、多人一致性统计和审核历史查询。
- 补充截图、演示视频和在线部署地址。
